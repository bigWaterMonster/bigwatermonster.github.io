浅析 URL

URI URI = URL + URN URI可被视为定位符（URL）,名称（URN）或两者兼备。统一资源名（URN）如同一个人的名称，而统一资源定位符（URL）代表一个人的住址。换言之，URN定义某事物的身份，而URL提供查找该事物的方法。 URL 例: https://xxxx.com:4321/A?name=helloWorld&password=123456#C 协议 + 域名 + 端口 + 路径 + 查询参数 + 锚点 协议: https 域名: xxxx.com:43 域名 端口: :4321 端口 路径: /A 查询参数: name=helloWorld&password=123456 锚点: #C IP 分, ipv4(32位) 123.123.123.123 每组10进制 ipv6(128位) 2001:0db8:86a3:08d3:1319:8a2e:0370:7344 每组16进制 主要作用使用户可以链接网络, 相当于一个门牌号, 根据门牌可以找到你，你有别人的ip也可以找到别人, 可分内网Ip和外网Ip, 常用ping命令测试ip地址直接是否可以通信 域名 ip地址由数字组成不方便记忆，所以有了域名更好的让人记忆，指向一个ip，可能一个ip同时有多个域名指向他。 如: baidu.com -> 1.2.3.4 分类 例 a.b.c.com com为顶级域名 c为2级域名 b为3级域名 以此类推a为4级之后每多一个点就多一级 b.c.com a.c.com 2个域名有相同顶级域名com和二级域名c.com, b.c.com 和 a.c.com为com或c.com的子域名 DNS 的作用 DNS 域名服务器 负责解析域名为ip地址
