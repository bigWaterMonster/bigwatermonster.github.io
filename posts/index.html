Js的诞生

Js诞生记 网景公司(Netscape) 1994年，网景公司（Netscape）发布了Navigator浏览器0.9版。这是历史上第一个比较成熟的网络浏览器。 为了使得浏览器可以与网页互动,网景公司指派Brendan Eich负责开发一种网页脚本语言。 Brendan Eich为了想要足够简单, 并且也没想到会像今天如此受欢迎, 花了10天就设计出了在JavaScript的第一版。(命名在JavaScript是为了蹭java的热度)。但是因为时间太过仓促所以为这个语言埋了很多的坑。 js十大设计缺陷 微软 JavaScript在浏览器推出后大获成功,所以微软不久后在自家ie浏览器上推出了Jscript与其竞争,并通过在windows上捆绑ie浏览器的形式逐步占有了大部分的时长份额。 ECMA 由于JavaScript和Jscript在2家浏览器之间互相不兼容,网景公司像ECMA（欧洲计算机制造商协会）提交语言标准。通过后以JavaScript语言为基础制定了ECMAScript标准规范。 es6 至此之后为了填补当初设计时造成的缺陷又为ECMAScript加入了很多新的功能。 es3 没有es4，直接从es3->es5 es5 es6 2015年 基本完善 之后每年一个版本命名为ES+推出年份:如ES2019 总结 Javascipt的诞生被他的作者嫌弃（虽然现在也有很多问题）,但最终经受了历史的考验,成为了现今浏览器最主流的脚本语言,相信他肯定还有更加完善的时候,值得学习。

浅析 URL

URI URI = URL + URN URI可被视为定位符（URL）,名称（URN）或两者兼备。统一资源名（URN）如同一个人的名称，而统一资源定位符（URL）代表一个人的住址。换言之，URN定义某事物的身份，而URL提供查找该事物的方法。 URL 例: https://xxxx.com:4321/A?name=helloWorld&password=123456#C 协议 + 域名 + 端口 + 路径 + 查询参数 + 锚点 协议: https 域名: xxxx.com:43 域名 端口: :4321 端口 路径: /A 查询参数: name=helloWorld&password=123456 锚点: #C IP IP地址分2类 ipv4(32位) 123.123.123.123 每组10进制 ipv6(128位) 2001:0db8:86a3:08d3:1319:8a2e:0370:7344 每组16进制 主要作用 使用户可以链接网络, 相当于一个门牌号, 根据门牌可以找到你，你有别人的ip也可以找到别人, 可分内网Ip和外网Ip, 常用ping命令测试ip地址直接是否可以通信 域名 ip地址由数字组成不方便记忆，所以有了域名更好的让人记忆，指向一个ip，可能一个ip同时有多个域名指向他。 如: baidu.com || 7k7k.com -> 1.2.3.4 分类 例 a.b.c.com com为顶级域名 c为2级域名 b为3级域名 以此类推a为4级之后每多一个点就多一级 b.
